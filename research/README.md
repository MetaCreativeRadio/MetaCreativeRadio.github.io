# リサーチ資料ディレクトリ

このディレクトリには、論構文作成のためのリサーチ資料を保存します。

## ファイル形式

各リサーチ資料は以下の形式でMarkdownファイルとして作成してください：

```markdown
# [概念名・理論名]

## 概要

[概念の簡潔な説明 - 2〜3段落程度]

## 詳細

### 歴史的背景

[概念の起源、発展の歴史]

### 核心的な考え方

[概念の中核となる思想・理論]

### 重要な論点

[議論すべき重要なポイント]

### 実践への示唆

[ビジネス・組織・クリエイティブへの応用]

## キーワード

- キーワード1: 説明
- キーワード2: 説明
- キーワード3: 説明

## 参考文献

- 書籍1
- 論文1
- Webリソース1

## メモ

[追加の考察、関連する概念など]
```

## 使い方

### 1. リサーチ資料の作成

```bash
# 例: U理論のリサーチ資料を作成
vim research/U理論.md
```

### 2. 論構文バンドルの準備

```bash
npm run prepare-research "U理論"
```

これにより、`essay-input/research-U理論-bundle.md` が生成されます。

### 3. Web版Claudeで論構文生成

生成されたバンドルファイルの内容をWeb版Claude（Sonnet 4.5）にコピペして、対話篇形式の論構文を生成します。

### 4. 生成された論構文の保存

```bash
npm run save-research "U理論"
```

生成された論構文を貼り付けて、`essays/research-U理論.md` として保存します。

## 対話篇の特徴

- **Opi（理論家役）**: リサーチ資料の内容を理論として紹介
- **タムカイ（実践家役）**: 自身の実践経験から理論を理解しようとする
- **自然な対話**: 完全創作された対話で、理論と実践を架橋

## リサーチ資料の例
