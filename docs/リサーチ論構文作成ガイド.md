# リサーチ論構文作成ガイド

このガイドでは、リサーチ資料から対話篇形式の論構文を作成する方法を説明します。

## 概要

**別途作成したリサーチ資料（レポート）** から、Opiとタムカイの対話篇形式の論構文を生成するワークフローです。

### 特徴

- **既存レポートを活用**: 事前に作成した学術理論・概念のリサーチ資料を使用
- **Opi（理論家役）**: リサーチ資料の内容を理論として紹介・解説
- **タムカイ（実践家役）**: 自身の実践経験（グラレコ、Purpose Carving等）から理論を理解
- **対話は完全創作**: リサーチ資料と話者特性に基づいて自然な対話を生成
- **理論と実践の架橋**: 抽象的な理論を具体的な実践に接続

## ワークフロー

### Step 1: リサーチ資料（レポート）の準備

事前に作成したリサーチ資料を `research/` ディレクトリに配置します。

**ファイル名**: `[概念名].md` （例: `二重過程理論.md`, `U理論.md`）

**リサーチ資料の形式**:
- Markdown形式（`.md`）
- 内容・構成は自由（既存のレポートをそのまま使用可）
- 学術理論、概念、思想などの調査・分析結果

**推奨される内容**（ガイドライン）:
- 概念の説明・定義
- 歴史的背景や起源
- 核心となる考え方・理論
- ビジネスやクリエイティブへの示唆
- 参考文献（あれば）

**配置例**:

```bash
# 既存のリサーチ資料をコピー
cp ~/Documents/二重過程理論レポート.md research/二重過程理論.md

# または新規作成
vim research/U理論.md
```

### Step 2: 論構文バンドルの準備

配置したリサーチ資料から、論構文作成用のバンドルファイルを生成します。

```bash
npm run prepare-research "[概念名]"
```

**例**:
```bash
npm run prepare-research "二重過程理論"
npm run prepare-research "U理論"
```

**生成されるファイル**: `essay-input/research-[概念名]-bundle.md`

**バンドルに含まれる内容**:
1. リサーチ論構文作成用プロンプト（対話形式で論構文を作る指示）
2. Opiの完全プロフィール（理論家役）
3. タムカイの完全プロフィール（実践家役）
4. **あなたが作成したリサーチ資料の内容**（`research/[概念名].md`）

### Step 3: Web版Claudeで論構文生成

1. 生成されたバンドルファイル（`essay-input/research-[概念名]-bundle.md`）を開く
2. **全内容をコピー**
3. **Web版Claude（Sonnet 4.5）**に貼り付け
4. Claudeが自動的に対話篇形式の論構文を生成

**生成される論構文の構造例**:

```markdown
# [概念名]への探求 - 理論と実践の対話

## 導入: 実践からの問い

[タムカイ]: 「最近、ワークショップで...」

[Opi]: 「それは面白い視点ですね...」

## 理論の展開: [概念]とは何か

[Opi]: 「[概念]というのは...」

[タムカイ]: 「なるほど。それって、僕がグラレコでやってる...」

## 実践との接続

[対話が続く]

## 注釈

[1] **[用語]**: [詳細な解説]
...
```

### Step 4: 生成された論構文の保存

```bash
npm run save-research "[概念名]"
```

**例**:
```bash
npm run save-research "U理論"
```

1. コマンドを実行
2. プロンプトが表示されたら、Web版Claudeで生成された論構文を**貼り付け**
3. **Ctrl+D** で入力完了
4. `essays/research-[概念名].md` として保存される

### Step 5: 確認とGitコミット

```bash
# 生成された論構文を確認
cat essays/research-U理論.md

# Gitにコミット
git add .
git commit -m "📝 リサーチ論構文追加: U理論"
git push
```

## 対話篇の品質を高めるコツ

### リサーチ資料（レポート）準備時

1. **概念の核心が含まれているか確認**: 最も重要な考え方が記述されているか
2. **歴史的文脈があるとベター**: 概念の起源や発展の説明があると対話が豊かに
3. **実践への示唆があるとベター**: ビジネス・組織への応用例があるとタムカイの反応が具体的に
4. **既存のレポートそのままでOK**: 完璧な構成でなくても、Claudeが対話形式に再構成します

### Web版Claudeでの生成時

- **プロンプトは変更不要**: バンドルファイルをそのまま貼り付け
- **生成が途中で止まったら**: 「続きをお願いします」と入力
- **対話が硬い場合**: 「もっと自然な会話にしてください」と追加指示

## よくある質問

### Q: リサーチ資料はどのくらい詳しく書けばいい？

A: 既存のレポートをそのまま使用できます。概要と核心的な考え方が含まれていれば、数段落〜複数ページまで自由です。Claudeが対話形式に再構成し、必要に応じて注釈で補完します。

### Q: 対話は本当に創作してもらえる？

A: はい。プロンプトで「対話は完全に創作してよい」と指示しています。リサーチ資料の内容と話者特性に基づいて、自然な対話が生成されます。

### Q: Opiとタムカイの役割分担は？

A:
- **Opi**: 理論を紹介・解説する役割（リサーチ資料ベース）
- **タムカイ**: 実践経験から理解しようとする役割（グラレコ、ワークショップ等）

### Q: ラジオエピソードの論構文との違いは？

A:
- **ラジオ版**: 実際の対話ログから再構成
- **リサーチ版**: リサーチ資料から対話を創作
- どちらも同じ話者（Opi & タムカイ）、同じ想定読者（ビジネスパーソン）

## トラブルシューティング

### エラー: リサーチ資料が見つかりません

```bash
エラー: リサーチ資料ファイルが見つかりません: research/[概念名].md
```

**解決**: `research/` ディレクトリに該当するMarkdownファイルを作成してください。

### 生成された論構文が短すぎる

**解決**: 元のリサーチ資料に詳細を追加して再生成するか、Web版Claudeで「もっと詳しく展開してください」と指示。

### 対話が不自然

**解決**: Web版Claudeで「もっと自然な会話調にしてください」と追加指示。
